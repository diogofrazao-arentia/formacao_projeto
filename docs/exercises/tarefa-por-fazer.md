# Tarefa por fazer

## Pedido inicial

A aplicacao permite criar e acompanhar tickets internos, mas falta uma forma de
registar contexto durante o acompanhamento.

Queremos conseguir adicionar comentarios aos tickets.

Ainda nao esta decidido:

- que informacao deve ter um comentario;
- onde os comentarios aparecem;
- como se adiciona um comentario;
- que validacoes sao necessarias;
- que erros devem ser tratados;
- que testes provam que a funcionalidade funciona.

Esta tarefa e intencionalmente vaga. Antes de implementar, transformar este
pedido numa spec funcional clara.

## Fluxo

Seguir o fluxo principal:

```text
../00-training-flow.md
```

Completar primeiro a spec:

```text
../specs/planned/FR6-comentarios-nos-tickets.md
```

Depois seguir a sequencia:

1. Iniciar.
2. Fazer spec.
3. Pedir plano tecnico.
4. Implementar.
5. Testar.
6. Atualizar docs.
7. Fazer commit.
8. Verificar CI.
